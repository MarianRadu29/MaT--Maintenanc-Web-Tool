1. Introduction

1.1 Purpose

This project provides an online scheduling and inventory management system for motorcycle, bicycle and scooter (electric or traditional) services. The purpose of the application is to:

  • Allow them to consult the available calendar and quickly book a time slot, filling out a form with dates, times and details about the technical problem, including attaching photos or videos.
  • Provide administrators with a control panel where they can approve or reject appointments, with personalized messages (e.g. "We don't have the necessary items in stock, come back in Z days") and add useful information such as estimated price or warranty.
  • Keep track of items stocks.
  • Import and export data in CSV formats about items and orders.

1.2 Document Conventions

This document follows the template for software requirements documentation according to the IEEE Software Requirements Specification.

1.3 Intended Audience and Reading Suggestions

  • Owners and administrators of two-wheeler service stations PFA/SRL contractors who manage repair shops for motorcycles, bicycles or scooters (electric or traditional). They need a centralized tool for scheduling orders, monitoring items stocks, as well as for quickly generating reports in CSV format.
  • Mechanics and service technicians Specialists who want immediate access to the details of each appointment (date, time, problem description, attached photos/video) and who want to update the status of interventions and the history of work directly from the web interface.
  • End customers (two wheels) Bicyclists, scooters and motorcyclists who want an intuitive, self-service appointment, with the possibility of attaching photos/video to describe the malfunctions and the first notifications by email of approval, rejection or price offer and warranty.
  • For software developers, it is recommended to read the sections on functional requirements, system features, and external interfaces to grasp the technical specifications and implementation details.

1.4 Product Scope

The Maintenance Web Tool (MaT) aims to streamline and automate the entire lifecycle of service appointments and inventory management for motorcycle, bicycle, and scooter repair shops. Its primary objectives are to provide customers with a real-time view of workshop availability, enable quick, multimedia-rich booking of time slots, and facilitate seamless communication between clients and administrators. For business owners, MaT offers an intuitive dashboard to approve or reject requests with tailored messages, estimate costs, manage warranty terms, and track parts stock levels and supplier orders. The system also supports flexible data import/export (CSV).By centralizing scheduling, communication, and inventory tracking, MaT enhances customer satisfaction, optimizes resource planning, and delivers actionable insights for continuous service-quality improvements.

2. Overall Description

2.1 Product Perspective

Maintenance Web Tool is an independent web system that can be installed on the service's own server and offers standardized data import and export (CSV). The application has a responsive frontend and a REST API backend. The system communicates with external SMTP email services for notifications. Through these mechanisms, the tool works both autonomously and as an integrated module in an existing software suite, ensuring a transparent and secure data exchange.

2.2 Product Functions

  • Allow users to register, log in, and manage their profiles.
  • Calendar view with real-time service availability.
  • Create, modify and cancel appointments with attached descriptions, photos and videos.
  • Automatic email notifications for confirmation, approval or rejection.
  • Administrator/Mechanic approval/rejection flow, with personalized messages and details (estimated price, warranty).
  • Parts inventory management.
  • Data import (inventory) in CSV formats
  • Data export related to customer orders CSV
  • Allow administrators to manage items and users.

2.3 User Classes and Characteristics

  • Client:
    • Variable technical level, from beginner to enthusiast.
    • Can register, log in, search for items, submit ratings and reviews, and participate in Q&A sections.
    • Intuitive interface, compatible with desktop and mobile.
    • Can attach multimedia files (photos, videos) to the scheduling form
    • Receive automatic email notifications regarding scheduling status
  • Mechanic:
    • Full rights: approve or reject appointments with personalized messages
  • Administrators:
    • Responsible for managing users and items.
    • Advanced knowledge of the operational and financial flow of the service.
    • Full rights: approve or reject appointments with personalized messages.
    • Manage parts stocks and place orders with suppliers.
    • Needs report exports in CSV, JSON and PDF formats for accounting and analysis
  • Guest Users:
    • Unregistered visitors who can browse the list of available time slots but they can't be programmed.
    • Potential to login/register in the app.

2.4 Operating Environment

The software will operate in a local environment (localhost) without a dedicated server. It is intended to run on standard personal computers with the following characteristics:

  • Hardware: Any modern PC capable of running a web browser and basic local server software.
  • OS: Compatible with Windows 10/11 (and maybe macOS and Linux distributions).
  • Software Components: Requires a local web server stack(Java - JDK, PostgresQL ) for running and testing the application.

2.5 Design and Implementation Constraints

  • No specific regulatory or corporate policies apply.
  • Security and performance requirements are minimal due to the local, non-production environment.

3. Application Interface

The Maintenance Web Tool’s interface has been crafted for maximum intuitiveness, clarity, and inclusivity—ensuring that every user can effortlessly browse the system, quickly locate what they need, and engage with its features without hesitation.

3.1 User Interfaces

  • Home Page: The home page welcomes users with a clear hero section and quick links to key functions (book appointment,calendar section).
  • Login Page: This page provides a simple, secure form for users to sign in with email and password. It includes client-side validation and a “Forgot Password?” link for account recovery.
  • Registration Page: Users can create a new account by entering their name, email, phone number, and choosing a password. Real-time checks ensure valid email format and strong password criteria.
  • Forget-Password Page: Clients enter their registered email to request a password reset link. The page confirms when the reset email has been dispatched.
  • Reset-Password Page: The reset form requires users to enter and confirm their new password. Password strength feedback guides them to choose a secure credential.
  • Calendar Page: An interactive calendar displays available time slots by day. Users click a slot to initiate appointment and see real-time availability updates.
  • Appointment page: This form collects vehicle details (type, model), a problem description, and allows drag-and-drop uploads of images or video. Inline help tips guide users through each field.
  • Profile page(info): The profile info section shows user details and account settings. Edit buttons let users update their info.
  • Profile page(appointment section): Lists all upcoming and past appointments with status badges. Users can cancel open appointments or confirm modifications requested by the admin.
  • Management Appointment page(admin): Admins see a sortable table of all service requests with filters for status. Action buttons open modals to approve with notes or send a custom rejection message.
  • Management inventory page(admin): Displays current stock levels and items catalog. Inline editing and low-stock alerts help admins reorder before supplies run out.
  • Data import/export page(admin): Provides tools to upload CSV files for batch imports and to download filtered reports as CSV. A field-mapping preview ensures data aligns correctly.
  • Users page(admin): Provides tools to upload CSV files for batch imports and to download filtered reports as CSV. A field-mapping preview ensures data aligns correctly.

  • The application will use HTML CSS for awesome design.

4. Characteristics of the application

4.1 Account Management

  • A user can register by choosing an email,a phone number, a password, first name, and last name. They can authenticate themselves by only needing the email and password.
  • A user can see all available time slots by day and can make an appoitment.
  • When a new user is created, their credentials are entered into the database. Similarly, when the user decides to modify their credentials, the new values are also updated in the database.
  • Authentication is required to access personalized features (e.g., submitting appointment form, managing profiles).
  • Session management ensures users remain logged in during their visit.
  • Users can log out securely at any time.
  • Administrators have separate authentication with elevated privileges.

4.2 Admin Section

  • The Admin Section is intended for users with administrator rights, offering comprehensive control over service operations including appointment approval or rejection with custom messages. From the dashboard, admins can manage inventory levels, and import or export data in CSV format.
  • When the admin change the status about all appointments,the updates are inserted into DB.Same about inventory items(price,level/stock,suppliers).

5. Other Nonfunctional Requirements

5.3 Security Requirements

  • Password Security: All user passwords are securely encrypted (Bcrypt) before being stored in the database to prevent unauthorized access.
  • Input Sanitization: All user input is thoroughly sanitized and validated before being stored in the database or displayed to users, protecting against malicious data.
  • SQL Injection Prevention: All database queries use prepared statements or parameterized queries to prevent SQL injection attacks.
  • Protection Against XSS: Output is properly escaped to prevent Cross-Site Scripting (XSS) attacks, ensuring that user-supplied content cannot execute scripts in the browser.

5.4 Software Quality Attributes

  • Usability: The system is designed to be easy to use, with a simple and intuitive interface suitable for all user types.
  • Simplicity: The project maintains a straightforward structure, making it easy to understand, maintain, and extend.
  • Reliability: Basic validation and error handling ensure the system performs consistently in a local environment.
  • Maintainability: The codebase is organized and documented to facilitate future updates or modifications.
