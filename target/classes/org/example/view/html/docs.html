2. Introduction

2.1 Purpose

This project provides an online scheduling and inventory management system for motorcycle, bicycle, and scooter services. It enables:

  • Quick booking of service appointments with date, time, and multimedia attachments;
  • Administrator approval/rejection of appointments with custom messages and additional details;
  • Inventory tracking and CSV import/export of items and orders.

2.2 Document Convention

Written in Markdown, this document follows the IEEE Software Requirements Specification template.

2.3 Target Audience

The platform is aimed at:

  • Service owners and administrators managing repair shops;
  • Mechanics and technicians needing appointment details;
  • End customers booking service slots with multimedia descriptions.

3. General Description

3.1 Product Perspective

MaT is an independent web system with a responsive frontend, REST API backend, SMTP integration, and multimedia storage.

3.2 Product Features

  • Calendar view of real-time availability
  • CRUD operations for appointments with attachments
  • Automated email notifications
  • Administrator approval flow
  • Inventory management
  • CSV import/export

3.3 User Classes and Characteristics

Client

  • Intuitive interface; desktop & mobile
  • Multimedia attachments
  • Email notifications

Administrator / Mechanic

  • Full control over appointments & inventory
  • CSV, JSON, PDF report exports

3.4 Operating Environment

Accessible in any modern browser supporting HTML5, CSS3, and JavaScript.

3.5 User Documentation

This SRS document and inline help tips guide end-users and administrators.

4. Application Interface

4.1 User Interface

Home Page

Hero section with quick links; responsive layout.

Login / Registration / Password Recovery

Secure forms with client-side validation.

Calendar & Appointment Pages

Interactive slots, drag-and-drop uploads.

Profile & Admin Pages

User info, appointment lists, admin tables, CSV tools.

4.2 Hardware Interface

Runs on any server; requires internet for SMTP.

4.3 Software Interface

Frontend (HTML/CSS/JS), RESTful HTTP API, JSON & multipart-form data.

4.4 Communication Interface

HTTP(S) transport; JSON payloads; form-data for media.

5. Characteristics of the Application

Account Management

Registration, authentication (email/password), appointment booking.

Admin Section

Appointment approvals, inventory updates, CSV import/export.

Session Management

JWT-based sessions, 1-day expiration, role-based access.

6. Technologies for Protection and Security

  • Password hashing with Bcrypt
  • JWT authorization
  • Prepared statements to prevent SQL injection
  • HTTPS & CORS configuration
  • Content Security Policy (CSP)
